## 📚🔥🚒 Цель проекта
Программа задумывалась для сохранения хоббийного и научно-технического контента, в основном зарубежного, на фоне тотальных блокировок доступа к информации в России. А также из интереса к программированию и автоматизации рутинных задач.

---

## ⚠️ Ограничения

> 🔥 **451saver бесполезен там, где всё уже сгорело**

Если доступ к YouTube или связанным сервисам в вашей сети ограничен — 451saver не поможет «пройти сквозь стену».

Внутри нет:

- VPN
- прокси
- обходов блокировок

Программа работает только с тем Интернетом, который у вас есть.

## ⚙️ Как это работает

451saver управляет следующими инструментами:

- [`yt-dlp`](https://github.com/yt-dlp/yt-dlp) - загрузка контента и метаданных с YouTube
- [`ffmpeg` / `ffprobe`](https://www.ffmpeg.org/) - наполнение MKV контейнера, метаданные, микширование аудио
- [`jq`](https://jqlang.org/) - парсинг JSON с YouTube
- [`mkvpropedit`](https://mkvtoolnix.download/doc/mkvpropedit.html) - настройка дорожек по умолчанию в MKV контейнере
- [`vot-cli-live`](https://github.com/fantomcheg/vot-cli-live) - загрузка ИИ озвучки и субтитров от Яндекса
- [`node`](https://nodejs.org/en) - служебный модуль для работы `yt-dlp` и `vot-cli-live`

👉 Пользователь взаимодействует только с меню — вся сложность скрыта

## 🚀 Режимы работы

### 🔹 1. Скачивание одного видео
Входные данные: ссылка

### 🔹 2. Пакетный режим
Входные данные: список ссылок из `.txt` файла

- 📄 `%имя_файла_со_списком%_log_errors_%дата_время%.txt` — создаётся лог ошибок
- 📄 `%имя_файла_со_списком%_log_queue_%дата_время%.txt` — создаётся лог оставшихся загрузок
- Если на протяжении всего списка не случилось сбоев, то в конце оба лога будут пусты
- В случае сбоя или прерывания работы эти логи можно использовать в качестве входных файлов для Пакетного режима

### 🔹 3. Пакетный режим (с прерыванием)
- То же, что и Пакетный, но с ручным подтверждением перехода к следующей загрузке
- Было полезно, когда требовалось проигрывать видео в браузере для активации ИИ-озвучки

### 🔹 4. Генерация списка видео канала
Входные данные: ссылка на канал

### 🔹 5. Пакетная проверка AI-озвучки
Входные данные: список ссылок из `.txt` файла
- Загрузка AI-озвучки и субтитров (RU + EN) от Яндекса
- Также полезно для более-менее быстрого понимания «наполненности» списка ссылок готовой озвучкой Яндекса

## 🛠 Настройки загрузки

### 📺 Видео

- выбор разрешения: **360p** / **480p** / **720p** / **1080p** / **1440p** / **2160p** / **Максимальное**

### 🧠 AI-озвучка

Если видео имеет хотя бы минимальные просмотры русскоязычной аудиторией с запросом ИИ-озвучки Яндекса, то загрузка этого перевода, как правило, происходит без проблем.

### 🧰 Форсирование языка

Предоставлен список из языков, перевод с которых поддерживается Яндексом. Если язык видео не определен, то загрузка озвучки может дать сбой. В этом случае можно установить английский язык, если известно, что весь целевой список загрузки, например, англоязычный. Если языки разные / контент без речи / озвучка не требуется, то функцию лучше отключить.

Доступно только для Пакетного режима. Также влияет на установку языка в свойствах звуковой дорожки.

### ⏸️ Пауза

Можно приостановить процесс, если AI-озвучка не найдена.
В этом случае следует:

1. Использовать расширение [voice-over-translation](https://github.com/ilyhalight/voice-over-translation)
2. Открыть видео в браузере  
3. Активировать перевод  
4. Скачать дорожку  
5. Поместить в рабочую папку

- 451saver автоматически продолжит обработку
- Данная функция всегда задействована в Одиночном режиме и опционально включается в Пакетном

## 🛠 Особенности

### 🔊 Аудио

При включении AI-озвучки готовый файл содержит:

- оригинальную дорожку
- чистую AI (RU)
- микшированную дорожку (по умолчанию для плеера)

---

### 💬 Субтитры

- русские (AI)
- английские (AI)
- по умолчанию для плеера все субтитры выключены

Сохраняются:

- в контейнер с видео
- в `.srt`

---

### 🖼️ Дополнительно загружаются

- обложка
- описание видео
- главы (если есть)

---

## 🛠 При запуске скрипта
При первом запуске скрипта происходит запрос Рабочей диреткории, Браузера и Профиля браузера. При повторном запуске спрашивается оставить ли текущие параметры или задать новые.

## 📁 Рабочая директория

Место, куда будет сохраняться контент

## 🌐 Настройка браузера

- выбрать браузер
- выбрать профиль браузера*

Поддерживаются:

- **Firefox**
- **Google Chrome**

⚠️ * Необходимый профиль подсвечен, но не исключено, что иногда нужно будет выбрать другой профиль

💡 Выбранные настройки хранятся в файле `config.txt` рядом со скриптом
